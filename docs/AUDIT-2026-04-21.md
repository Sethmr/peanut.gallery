## A2 · Post-audit bug class surfaced separately — TDZ in sidepanel.js

A bug class NOT caught by this audit pass turned up the same day (2026-04-21) and is worth recording here so future audits remember to scan for it.

**What it is:** `sidepanel.js` is 3,500+ lines in one file. When a new large top-level code block is added and references `const` DOM bindings declared later in the file, accessing those bindings at parse time throws `ReferenceError` from the temporal dead zone. That halts the entire script — every `addEventListener` past the throw point never attaches, and the whole panel looks visually correct but no button is clickable. `<a href>` links still work because browsers navigate those natively without JS.

**Instances shipped-and-fixed this cycle:**

- [#106](https://github.com/Sethmr/peanut.gallery/pull/106) — v1.9 subscription event-listener block at line ~670 referenced `const backendModeSegmented` / `subscriptionKeyInput` / etc. declared at line ~828. Fix: hoisted the 12 subscription DOM refs up to line ~534.
- [#112](https://github.com/Sethmr/peanut.gallery/pull/112) — top-level `for` loop at line ~851 iterated over `[deepgramKeyInput, anthropicKeyInput, xaiKeyInput]`, all declared at line ~964. Fix: reach through `document.getElementById(id)` inline instead.

**Recognition pattern** (on-panel visible):
- DOM renders ✓
- `:hover` works on buttons ✓
- `<a href>` links work ✓
- `<button>` clicks do nothing ✗

**Diagnostic that cracked it:** visible on-panel diag strip (PRs [#108](https://github.com/Sethmr/peanut.gallery/pull/108)–[#111](https://github.com/Sethmr/peanut.gallery/pull/111)) showing load-timestamp + click-counter + last-phase-reached + sticky error at `z-index: 99999, pointer-events: none`. `updatePhase()` breadcrumbs at every major load site made the halt point visible without DevTools. Stripped once root cause was fixed ([#113](https://github.com/Sethmr/peanut.gallery/pull/113)). Full post-mortem: [`DEBUGGING.md § ISSUE-010`](DEBUGGING.md#issue-010-sidepanel-tdz--nothing-tappable-after-a-new-top-level-block). Lesson saved to Claude's auto-memory (`feedback_sidepanel_tdz_lesson.md`) so future AI work on the file doesn't repeat it.

**What's kept as a safety rail (post-fix):**
- Minimal `window.addEventListener("error")` + `unhandledrejection` handlers at the top of sidepanel.js — log any future uncaught throw to console as `[PG:sp fatal]`.
- Defensive tutorial-overlay force-hide at init (`_initTutorialOverlay`). Insurance against the stuck-backdrop class of failure.

**Connects to this audit's finding C2** — decomposing `sidepanel.js` into ES6 modules would make this class of bug structurally impossible (per-module scope, no cross-module forward refs). Elevates C2's post-v2.0 priority: the refactor isn't just a code-quality win, it's a class-of-bug eliminator.

## C · High — deferred

Non-blocking but worth tickets before v2.0 launch.

### C1 · Persona-engine timeout recovery lacks a telemetry surface (finding #4)

Severity: High · Cost: M (~4 hours).
`firePersona` has `AbortSignal.timeout(25_000)` but no explicit telemetry event if the signal fires mid-call. Design principle #1 says a committed UI surface (force-react, producer slot) always lands visible content — the force-react fallback path honors this, but if a timeout fires mid-stream, we'd want a `persona_timeout_fallback` telemetry event so v1.7 GA tuning has a signal.

**Ticket suggestion:** "Persona timeout → fallback telemetry + 'took too long' subtle UI hint." Emit `persona_fallback_fired` with `reason: "timeout"` in `lib/persona-engine.ts`; sidepanel surfaces a small muted suffix on the fallback fire.

### C2 · sidepanel.js is 3.5k lines with no layer separation (finding #3)

Severity: High · Cost: L (2–3 days, post-v2.0).
Single monolithic file for DOM, event delegation, persona rendering, feed state, drawer control, settings persistence, empty-state logic. Not broken, but compounding complexity as new interactions land (v1.6 menu + pin + votes added ~600 lines).

**Ticket suggestion:** "Post-v2.0 — decompose sidepanel.js into ES6 classes." `PersonaAvatarManager`, `FeedEntryRenderer`, `DrawerController`, `SettingsPersistence`. Do NOT block v2.0 ship; queue for v2.x maintenance window.

### C3 · Empty-state variants — finding recheck false-positive

Severity: was marked High · Cost: 0.
Audit flagged "wire_quiet" and "no_claims" variants as missing CTAs. Neither variant exists in [`EMPTY_STATE_VARIANTS`](../extension/sidepanel.js) at [sidepanel.js:1137](../extension/sidepanel.js). Actual variants are `idle` (intentionally CTA-less — "nothing happening yet"), `needs-keys`, `unreachable`, `mic-denied`, `no-pack` — last four all have CTAs. Closing as not-actionable; agent hallucinated variant names.

### C4 · Unstable-tail heuristic is defined in v2 module but called from the route (finding #6)

Severity: High · Cost: S (~2 hours).
[`lib/director-llm-v2.ts`](../lib/director-llm-v2.ts) defines `computeUnstableTailLen`; [`app/api/transcribe/route.ts`](../app/api/transcribe/route.ts) calls it. If the heuristic definition is tuned in v3 without updating the route contract, they drift.

**Ticket suggestion:** "Extract `computeUnstableTailLen` + other Director utilities into `lib/director-shared.ts`."

Severity: Low · Cost: XS · **Audit was stale** (confirmed 2026-04-22).

const rateParsed = Number.isFinite(rate) ? Math.round(rate) : 5;
const rateClamped = Math.max(1, Math.min(10, rateParsed));

const rawSensitivity = (req.headers.get("X-Sensitivity") || "").toLowerCase();
