## 1. Goals, non-goals, done-definition

### Goals

1. LLM-assisted routing for the Director — one short Claude Haiku call per tick decides who speaks, with a one-sentence rationale flowing into the debug panel.
2. Hard 400ms budget on the LLM call; any overrun degrades cleanly to the rule-based scorer that has shipped since v1.2.
3. Observable rollout: every tick the flag is on emits enough telemetry to answer "is the LLM actually helping?" before flipping defaults.
4. No breaking wire changes. Older extensions connected to a v1.5 backend keep working; newer extensions connected to a v1.4 backend keep working.

### Non-goals

- Changing cascade probabilities, cooldown math, or recency penalties. The LLM picks the primary; every other Director invariant stays pinned.
- Replacing the rule-based scorer. It remains the safety floor forever.
- Per-pack LLM tuning. The routing prompt is pack-agnostic by design; pack-specific overrides can land in v1.5.x if the data says they help.
- Voice / TTS (v1.6). Clip-share (v1.6). 3D bobbleheads (v2.0). None of that touches the Director.
- Streaming the routing response. A ~30-token JSON payload is simpler and already well under the 400ms budget on Haiku.

## 2. Feature flag posture for the tag

**Decision for v1.5.0 launch:** explicit opt-in only. `ENABLE_SMART_DIRECTOR=true` in the environment is required. No auto-on when `ANTHROPIC_API_KEY` is present.

**Why:**

- Each tick costs one extra Haiku call. That is cheap in isolation but compounds for self-hosters who haven't tuned their Anthropic quota and may leave a session running for hours.
- We do not yet know the agreement rate or the LLM's true value-add. The canary (step in §4) is what generates the data that justifies flipping the default.
- Failing safe is "behave exactly like v1.4" — a release that already shipped. The floor is known-good.
- The flag is a single line in `.env.local`. Discoverability is solved by documenting it in both self-host guides; friction is basically zero.

**Revisit in v1.5.1:** if canary data shows `director_v2_compare.agreed` averaging ≥ 60% with p95 `llmElapsedMs` < 350 ms, flip the default to auto-on-when-Anthropic-key-present. Keep the flag as an explicit `ENABLE_SMART_DIRECTOR=false` kill switch.

## 4. Canary checklist

> **Note (2026-04-20):** this checklist no longer gates the v1.5.0 tag — v1.5.0 shipped without it. It now gates the **server flag flip** (whether to default `ENABLE_SMART_DIRECTOR=true` in a future release, and whether the data supports folding the scaffold straight into v1.7 GA vs. a v1.5.2 half-step).

After deploying with `ENABLE_SMART_DIRECTOR=true` for ≥ 48 hours of real traffic:

1. Pull all `director_v2_compare` events for the period from `logs/pipeline-debug.jsonl`.
2. Compute:
   - **Agreement rate** = `count(agreed=true) / count(all)`. Target ≥ 0.55. Lower means the LLM is adding a lot of variance — read 20 random disagreement transcripts before declaring that good.
   - **Override rate** = `count(overrode=true) / count(source='llm')`. Target 0.20 – 0.45. Below 0.20 means the LLM is basically agreeing with the rule-based scorer all the time (no value-add). Above 0.45 means the LLM is picking aggressively different personas — may be good, may be chaos; sanity-check by reading transcripts.
   - **p50 / p95 `llmElapsedMs`** for `source='llm'` rows. p95 must be < 350 ms. If p95 creeps toward 400 we are timing out too often, and `count(llmTimedOut=true) / count(all)` should also spike.
   - **Timeout rate** = `count(llmTimedOut=true) / count(all)`. Target < 0.05. Above 0.10 means the 400 ms budget is wrong for real Haiku latency and we should either bump the budget or accept rule-only fallback on 10%+ of ticks.
3. Sanity-read 20 `director_v2_compare` rows where `overrode=true`. Does the LLM's pick make sense given the transcript? Does the rationale read like a human producer's note?
4. Spot-check the debug panel in the side panel. `LLM` badge should appear on most rows during an active session. If it's `RULE` on 100% of rows, the flag isn't on or the key isn't present.

If the numbers fall outside the target bands, do NOT default the flag on. File the findings under `docs/SESSION-NOTES-<date>.md` and either adjust the prompt in `lib/director-llm.ts` or pull the feature and re-roadmap.

## 5. Rollback lever

Single line: set `ENABLE_SMART_DIRECTOR=false` (or unset it) and redeploy. Behavior reverts to v1.4 rule-based routing instantly on the next director tick. No session needs to be restarted; the flag is read per tick.

There is no stateful carryover — the Director's cooldown map, recent firings, and dry-spell counters are identical between the two modes. Flipping the flag mid-session produces a seamless transition.

## 6. Known risks

- **Prompt brittleness.** A bad routing prompt could consistently favor one persona slot. Mitigation: the 4 validated archetype ids force the model to pick one of four. If one slot dominates, we'll see it in `director_v2_compare` and can iterate on the prompt without a release cut.
- **Rationale drift.** The model might return a rationale that references a named persona ("pick Baba because…") in a TWiST session. Benign — the slot id routing is what's load-bearing. Rationale is for the debug panel.
- **Cost surprise for self-hosters.** Hosted prod absorbs the Haiku cost under existing quotas. Self-hosters opting in pay their own Haiku tier. Documented in SELF-HOST-INSTALL.md; can't be mitigated further without defaulting off (which is what we're doing).
- **Flag plus key race on session startup.** The route reads the flag per tick, so a session that started before the flag was enabled will pick it up on its next tick with no restart. No special handling needed.
- **Hosted demo-key fallback.** When the extension posts without an Anthropic header, the hosted backend falls back to its server-side demo key. Smart Director honors the same key resolution as persona firing — if the fallback succeeds for Haiku, it succeeds for routing. No new auth path.
- **yt-dlp path (web app / `/watch`).** The Smart Director doesn't care which transcription source the session is on. If yt-dlp is happy, Deepgram is happy, and Anthropic is reachable, the feature works identically on both extension and web-app paths.
